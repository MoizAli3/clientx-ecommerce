Generate a professional time and cost estimate for a project or feature.

Ask for:
1. What needs to be built (description or requirements doc)
2. Tech stack (if known)
3. Your hourly rate (default: $25/hr for Pakistani market, $50/hr for international)

Breakdown estimate into phases:

| Phase | Tasks | Hours | Cost |
|-------|-------|-------|------|
| Discovery & Setup | Requirements, DB design, boilerplate | X hrs | $X |
| Core Features | Main functionality | X hrs | $X |
| Auth & Security | Login, roles, API security | X hrs | $X |
| UI/UX | Frontend, responsive design | X hrs | $X |
| Testing & QA | Unit tests, manual testing, bug fixes | X hrs | $X |
| Deployment | CI/CD, production setup, docs | X hrs | $X |
| **TOTAL** | | **X hrs** | **$X** |

Add:
- Buffer: +20% for scope creep (always happens)
- Revisions: 2 rounds included, $X/hr after that
- Maintenance: optional $X/month

Delivery timeline: X weeks (assuming X hrs/day)

Payment terms recommendation:
- 50% upfront, 50% on delivery (for new clients)
- 30% upfront, 40% mid, 30% delivery (for large projects)

Be honest — underestimating kills projects. Better to over-deliver on time.
